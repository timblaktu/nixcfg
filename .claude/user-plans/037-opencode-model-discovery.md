# Plan 037: Automated CCv2/Bedrock Model Discovery for OpenCode

## Context

CCv2 (ai-proxy) and Bedrock proxy endpoints change models frequently — 7 new models appeared in 6 weeks (2026-04-04 to 2026-05-15). Currently, adding models requires editing Nix config → rebuild → switch. This plan adds a pre-flight discovery mechanism to the opencode wrapper that queries `/v1/models` endpoints and injects discovered models via `OPENCODE_CONFIG_CONTENT` (an official opencode env var with highest merge precedence).

## Design

```
[Nix-generated opencode.json]    ← static baseline (provider URLs, auth, known models with metadata)
         +
[~/.cache/opencode-discovery/]   ← cached /v1/models responses per provider
         ↓
[wrapper merges via jq]          ← only adds NEW models not in static config
         ↓
[OPENCODE_CONFIG_CONTENT env]    ← highest-precedence config (deep-merged by opencode)
         ↓
[exec opencode]
```

**Key properties:**
- Nix-defined models with custom attributes (modalities, display names) are never overwritten
- Discovery only ADDS models missing from static config
- 30-min cache TTL avoids API calls on every launch
- 3-second timeout prevents blocking if API is down
- Graceful fallback: if discovery fails, static config is used as-is

### T1: Discovery script package

**New file**: `modules/programs/opencode/_hm/model-discovery.nix`

A `writeShellApplication` package: `opencode-discover-models`

**Interface:**
```
opencode-discover-models \
  --provider-id bedrock \
  --base-url "https://proxy.example.com/api/v1" \
  --api-key-env BEDROCK_API_TOKEN \
  --cache-dir ~/.cache/opencode-discovery/work \
  --static-models "model-a,model-b,model-c"
```

**Behavior:**
1. Read API token from named env var (`$BEDROCK_API_TOKEN`)
2. `curl -sf --max-time 5` to `$BASE_URL/models`
3. Parse `.data[].id` from response (standard OpenAI format)
4. Filter out IDs present in `--static-models` (Nix-managed)
5. Emit minimal model entries: `{ "<id>": { "name": "<id>" } }`
6. Write `{ "provider": { "<provider-id>": { "models": { ... } } } }` to `$CACHE_DIR/<provider-id>.json` via atomic rename
7. On failure: log warning to stderr, leave existing cache untouched
8. Log retired models (in static but missing from API) to stderr as informational

**Dependencies**: `curl`, `jq` (in `runtimeInputs`)

**DoD**: Script runs standalone against live endpoints, produces valid JSON, handles failures gracefully.

### T2: Option schema

**File**: `modules/programs/opencode/opencode.nix` (account submodule options, ~line 375-470)

Add under each account:
```nix
discovery = {
  enable = mkEnableOption "automatic model discovery from /v1/models endpoints";
  cacheTtlMinutes = mkOption {
    type = types.int;
    default = 30;
    description = "Cache TTL before re-querying endpoints (minutes)";
  };
  timeoutSeconds = mkOption {
    type = types.int;
    default = 3;
    description = "Max seconds to wait for discovery before launching opencode";
  };
};
```

Discovery auto-detects which providers to query: any provider with `options.baseURL` set AND an `options.apiKey` matching `{env:*}` pattern.

### T3: Wrapper integration

**File**: `modules/programs/opencode/opencode.nix` (in `mkAccountScript`, ~line 1091-1147)

When `discovery.enable = true`, inject between token fetch and `exec opencode`:

```bash
# --- Model discovery ---
_OC_DISC_DIR="$HOME/.cache/opencode-discovery/${accountName}"
mkdir -p "$_OC_DISC_DIR"

# Per-provider discovery (background, with timeout)
_oc_disc_pids=()
for _frag in "$_OC_DISC_DIR"/*.refresh; do
  # Each .refresh file is generated by Nix with provider params
  source "$_frag" &
  _oc_disc_pids+=($!)
done

# Wait with timeout
for _pid in "${_oc_disc_pids[@]}"; do
  timeout ${timeoutSeconds} tail --pid=$_pid -f /dev/null 2>/dev/null || true
done

# Merge cached fragments into OPENCODE_CONFIG_CONTENT
if compgen -G "$_OC_DISC_DIR/*.json" >/dev/null 2>&1; then
  export OPENCODE_CONFIG_CONTENT
  OPENCODE_CONFIG_CONTENT=$(jq -s 'reduce .[] as $item ({}; . * $item)' \
    "$_OC_DISC_DIR"/*.json 2>/dev/null || echo '{}')
fi
```

The `.refresh` files are Nix-generated per discoverable provider, containing:
```bash
# bedrock.refresh (generated at build time)
opencode-discover-models \
  --provider-id bedrock \
  --base-url "https://ai-platform-bedrockapis.d-dp.nextcloud.aero/api/v1" \
  --api-key-env BEDROCK_API_TOKEN \
  --cache-dir "$_OC_DISC_DIR" \
  --static-models "us.anthropic.claude-opus-4-6-v1,us.anthropic.claude-sonnet-4-6,..."
```

Static model lists are computed at Nix eval time from `cfg.provider.<id>.models`.

**DoD**: Generated wrapper script contains discovery snippet. Inspect with `cat $(which opencodework)`.
